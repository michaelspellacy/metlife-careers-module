CSS

The animation is handled with CSS3. CSS3 animation is well supported and much easier to implement than with traditional methods, such as jQuery or Dojo. For those browsers that do not support animations, the last frame will be shown. This is expected behavior. The entire experience will degrade gracefully on older, less supported browsers.

Simply apply the following link to the <head> of your document. Preferably after all other CSS links on the page.

Code

<link href="css/profiles-4.3.css" rel="stylesheet"/>

View: profiles-4.3.css

Internet Explorer 8

Internet Explorer 8 will now require an additional CSS file. This should be placed in the head of your document, after profile-4.3.css has been referenced. This specifically targets Internet Explorer 8, so be sure to include the link reference in a conditional comment, like so:

Code

<!--[if IE 8]><link href="css/ie8.css" rel="stylesheet"/><![endif]-->

View: ie8.css

Sass

While the CSS above may be edited freely in the css directory after handoff, Sass (Syntactically Awesome Style Sheets) was used in the creation of all style sheets. The source files may be found in the build/sass directory and are compiled to the css directory in the root. Sass is not required to run this module in any of your environments. It has merely been provided here as a convenience for those developers that do write CSS using a preprocessor. If you do not use a CSS preprocessor, feel free to skip to the next section.

Sass Dependencies

In order to make writing CSS vendor prefixes easier, this configuration relies on Autoprefixer. You must install this gem before running compass watch within your project directory. To install Autoprefixer, issue the following command in terminal or command prompt:

gem install autoprefixer-rails

JavaScript

Two scripts are needed for the Career Path module to work properly:

Modernizr (modernizr-2.8.3-custom.js)

A custom version of Modernizr has been created to handle animation fallback. Modernizr leverages feature detection and works with the CSS that has been written. If CSS animation is not supported, then adequate fallback methods for older browsers, like Internet Explorer 8 and 9, can be built in.

This file should be placed within the <head> of your document before any other scripts or links.

Code

 <script src="js/modernizr-2.8.3-custom.js"></script>

View: modernizr-2.8.3-custom.js

Profile (profiles-4.3.js)

This is the custom functionality created for the navigation to toggle between each profile. It is written in pure JavaScript and should not interfere with any existing frameworks, such as Dojo.

This file should be placed at the bottom of the document, before the closing <body> element.

Code

 <script src="js/profiles-4.3.js"></script>

View: profiles-4.3.js

iframe Version

If integration proves to be difficult due to conflicts between Metlif's CSS and the CSS provided by TMP Worldwide, an alternative way to implement the profile would be to load it within an iframe. This seperates both code bases and eliminates conflicts within the cascade.

The module as it will appear within an iframe.

Code

 <iframe src="iframe-source.html" width="893" height="382" frameborder="0" scrolling="no" title="Metlife Career Profiles"></iframe>

To implement, simply add the iframe element to the parent page and point the src attribute to the location of the source file (iframe-source.html).

Note: Paths to assets that are loaded within iframe-source.html may need to be altered by Metlife's development team to reflect the configuration of their careers site.
